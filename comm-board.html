Bulletin Board Turbo — Internal Troubleshooting Manual

Hardware: Raspberry Pi 4 Software: LKT Signage on Debian Audience: Internal Support Staff Version 1.1 — March 2026
1 Overview

The Bulletin Board Turbo is a digital bulletin board powered by a Raspberry Pi 4 running LKT Signage on the Debian operating system. It displays content such as announcements, schedules, and media on any connected TV or monitor via HDMI.

The device only requires three things to function:

  • A power source (supplied power adapter)
  • An HDMI connection to a TV or monitor
  • An internet connection (Wi-Fi or Ethernet)

3 How the Device Works
  • Once powered on and connected to a TV via HDMI, a sample image should appear on the screen after booting.
  • Content is managed remotely through the LKT Signage dashboard.
  • The device must be connected to the internet to receive and display updated content.
Note on Power The device must be powered using the supplied power adapter. Powering from a TV's USB port does not supply adequate power and will cause the device to shut off intermittently.
4 Using the LKT Signage Dashboard

Accessible at lktsignage.com — used to manage all content displayed on the Communication Boards.

4.1 Assets
  • All content (videos, images, documents, etc.) is referred to as an asset and must be uploaded in the Assets section.
  • When uploading, you can assign it to a category to make it easier to find and filter later.
  • You can set the duration — how long each image or slide stays on screen.
  • You can also set a start and/or end date to schedule when an asset will be active.
4.2 Playlists

Once an asset is uploaded, it must be assigned to a playlist. The following playlists have been created:

  • All Stores — content assigned here will play on all store boards.
  • Individual store playlists — content assigned here will only play on that specific store's board.
4.3 Groups
  • Playlists are assigned to groups. A group can have multiple playlists assigned to it.
  • Groups are where you manage device settings such as resolution and screen orientation.
  • Each player is assigned to a group.
4.4 Deploying Changes
  • After uploading assets and assigning them to playlists and groups, you must deploy your changes for them to take effect.
  • To deploy all groups at once: click Groups, then click the Deploy All button in the top right corner.
  • To deploy a single group: click Groups, select the group, and deploy it individually.
If you forget to deploy, the system will automatically deploy all changes at 10:00 AM every day.
4.5 Checking Player Status
  • To see whether a board is online, click Players — each player has a status indicator showing whether it is online or offline.
  • To see what is currently being displayed on a board, click Players, then click the >_ button next to that player. This opens the Pi Shell, which shows a live snapshot of the screen along with playlist controls and other options.
5 Troubleshooting
5.1 Before You Begin

Verify the following before addressing any specific issue:

  • The supplied power cable is securely connected to the device and a wall outlet — not a TV USB port.
  • The red power light on the unit is on.
  • The HDMI cable is firmly connected between the device and the TV.
  • The TV is powered on and set to the correct HDMI input.
  • The MicroSD card is fully inserted into the slot on the edge of the device.
  • The TV volume is turned up if the content includes audio.
5.2 Device Not Booting / OS Not Loading

Symptoms:

Stuck on "cookie" loading screen Rainbow block on screen Scrolling text on screen Blinking cursor or error message Black screen after powering on Sample image never appears
Important These issues occur before the operating system fully loads and as a result are not affected by network issues. Do not troubleshoot Wi-Fi or the access point for these issues.
5.2.1 Power Cycle the Device
  1. Unplug the power cable from the device.
  2. Wait 30 seconds, then plug it back in.
  3. Wait up to 60 seconds for the device to fully boot.
If the device boots normally, no further action needed. Note the incident in case it recurs.
If the device still does not boot, continue to 5.2.2.
5.2.2 Check the SD Card
  1. Power off the device by unplugging the power cable.
  2. Locate the MicroSD card slot on the edge of the device.
  3. Gently slide the SD card fully into the slot until it feels secure.
  4. Reconnect the power cable and wait up to 60 seconds to boot.
If the card was loose, the device should now boot and display a sample image.
If the card appeared secure, continue to 5.2.3.
5.2.3 Inspect the SD Card for Damage
  1. Power off and remove the SD card by sliding it out.
  2. Visually inspect the card for cracks or damage to the gold contacts.
  3. Re-insert the card firmly and power the device back on.
If the card is visibly damaged, the SD card needs to be replaced or the device should be swapped out.
If the card looks fine, continue to 5.2.4.
5.2.4 Verify the Power Supply

The Raspberry Pi 4 requires a 5V / 3A USB-C power supply.

  1. Confirm the device is plugged into a wall outlet using the supplied power adapter.
  2. If a different adapter or cable is in use, switch back to the supplied one.
  3. Power the device on and wait up to 60 seconds.
If the correct adapter is in use and the issue persists, the device should be replaced.
5.3 Device Won't Power On
Red power light is off No lights on device Completely black screen
  1. Confirm the power outlet works by plugging in another device.
  2. Test the USB-C power cable by using it to charge another device such as a phone or tablet — if it does not charge, the cable may be faulty and should be replaced.
  3. Make sure the USB-C power cable is firmly connected to the Raspberry Pi.
  4. Confirm the device is using the supplied power adapter plugged into a wall outlet.
  5. Try a different wall outlet if available.
If the red light still does not come on, the power supply or board is likely faulty and the device should be replaced.
5.4 Display Problems
Screen shows nothing "No signal" message Distorted or flickering image
  1. Confirm the HDMI cable is securely connected at both ends.
  2. Check that the TV is set to the correct input/source (e.g., HDMI 1, HDMI 2).
  3. If the correct input shows no signal, try switching to a different HDMI input on the TV and moving the cable to that port.
  4. Try a different Micro-HDMI to HDMI cable if one is available. Note that this cable type is not commonly available — if a spare is not on hand, skip this step.
  5. Power cycle the device (unplug power, wait 30 seconds, reconnect).
  6. Connect the device to a different TV or monitor if possible to determine whether the problem is the device or the display. If no other TV or monitor is available, the device should be replaced.
5.5 Content Not Updating
Old content still displaying Scheduled content not appearing
  1. Confirm the device has an active internet connection (see 5.6).
  2. Log in to the LKT Signage dashboard and verify the correct content or playlist is scheduled and published.
  3. Check whether the device shows as online in the dashboard.
  4. If online but content hasn't refreshed, manually deploy the group from the Groups page.
  5. If the device shows as offline, resolve the network connection first, then recheck.
5.6 Connectivity Problems
Shows offline in dashboard Content not syncing Only sample image showing
  1. Confirm the internet connection is active at the device's location.
  2. If using Wi-Fi: Follow the steps in 5.6.1 to configure or re-enter Wi-Fi credentials.
  3. If using Ethernet: Check that the cable is firmly connected to both the Raspberry Pi and the router or network switch.
  4. Power cycle the device — this restarts the network connection as well.
  5. Log into the LKT Signage dashboard to confirm the device comes back online within a few minutes.
5.6.1 How to Connect to Wi-Fi
A USB keyboard is required for this process.
  1. Plug a USB keyboard into any available USB port on the device.
  2. Press F7 to bring up the settings page.
  3. Use the arrow keys to highlight WiFi Config and press Enter.
  4. Use the arrow keys to select the country and press Enter.
  5. Type in the network name (SSID) and press Enter.
  6. Type in the network password and press Enter.
  7. When asked if the network is hidden, answer accordingly and press Enter.
  8. The configuration will be saved and you will be returned to the settings screen.
  9. Press Tab to highlight Finish, then press Enter to restart the device.
After rebooting, check the LKT Signage dashboard to confirm the device is showing as online.
5.7 F7 Key Not Responding (Keymap Issue)
F7 does nothing Settings page won't open

Cause: This can occur when a system update overwrites the custom keymap configuration on the device.

The keymap can be restored by running a command through the LKT Signage dashboard. If you are unsure how to do this, please ask Jordan.

  1. Log in to lktsignage.com and click Players.
  2. Click the >_ button next to the affected player to open the Pi Shell.
  3. Paste the following command as a single line and click Execute:
sudo rm /usr/share/X11/xkb/keycodes/evdev /home/pi/piSignagePro/misc/stretch/lxde-rc.xml ; sudo wget -O /usr/share/X11/xkb/keycodes/evdev https://raw.githubusercontent.com/ninjordan/lktsignageupdates/main/evdev ; sudo wget -O /home/pi/piSignagePro/misc/stretch/lxde-rc.xml https://raw.githubusercontent.com/ninjordan/lktsignageupdates/main/lxde-rc.xml ; reboot
The device will reboot automatically. Once it comes back online, the F7 key should respond normally.
5.7.1 If the Device is Offline
  1. Connect an Ethernet cable from the device to any available port on a router or network switch.
  2. The device only needs power and Ethernet — it does not need to be connected to a TV or monitor.
  3. Wait a minute or two, then check the Players page in the LKT Signage dashboard.
  4. Once the device shows as online, follow the steps in 5.7 to run the command through the Pi Shell.
5.8 Device Remains Offline in Dashboard After Network Connection
Powers on, shows old content Offline in dashboard Won't connect via Wi-Fi or Ethernet
  1. Confirm Wi-Fi has been configured via F7 or that an Ethernet cable is securely connected.
  2. Power cycle the device and wait a few minutes for it to come back online.
  3. Check the Players page in the LKT Signage dashboard to see if the status indicator has changed.
If the device still shows as offline after confirming network connectivity and restarting, the device should be replaced.
